# Political Research Project - Claude Guidelines

## Project Purpose
Personal research project for understanding political history, authoritarian regimes, and historical parallels.

## Research Permissions
- Web searches for historical information: APPROVED
- Web searches for political research: APPROVED
- Web searches for academic/educational content: APPROVED

## Workflow
- Use web search freely for historical research
- Create detailed, sourced notes in Obsidian format
- Always cite sources with links
- Distinguish facts from analysis

## Note Structure
- `/facts/` - Sourced factual claims
- `/analysis/` - Personal interpretations (clearly labeled)
- `/sources/` - Source registry
- `/comparisons/` - Historical comparisons
- `/templates/` - Note templates

## Quality Standards
- Every fact must have at least one source
- Distinguish correlation from causation
- Label relationship types explicitly
- Note confidence levels

## Git Workflow
- **Always use feature branches** — never commit directly to `main`
- Branch naming: `update/<short-description>` (e.g., `update/april-1-3-developments`)
- Merge to `main` after commit is clean
- Commit messages: short title + bullet list of key changes
- Co-author line: `Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>`
- When updating the tracking file, auto-append and commit per the established pattern ("assume I just want you to update the file as we go")
